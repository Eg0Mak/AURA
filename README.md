## О проекте

**AURA — Augmented Universal Retrieval Assistant** — гибридная RAG-система для поиска релевантных фрагментов в корпусе документов и генерации ответа на основе найденного контекста.

Проект разработан в рамках хакатона **«Альфа-Будущее» от Альфа-Банка**.

Система объединяет:

* семантический поиск по эмбеддингам;
* лексический поиск с использованием TF-IDF;
* гибридное объединение результатов;
* Cross-Encoder reranking;
* генерацию ответа на основе выбранных документов;
* пользовательский интерфейс на Gradio.

Основная идея AURA — совместить преимущества плотного и разреженного поиска, а затем уточнить порядок найденных документов с помощью более точной модели ранжирования.

## Основные компоненты

### Data Preprocessing

Перед индексацией документы проходят предварительную обработку.

На этом этапе выполняются:

* очистка текста;
* удаление лишних символов;
* нормализация строк;
* подготовка данных к разбиению на чанки.

---

### Chunking

Документы разбиваются на небольшие фрагменты — чанки.

Chunking позволяет:

* находить конкретные смысловые фрагменты;
* уменьшить объём нерелевантного контекста;
* не передавать в LLM целые документы;
* повысить точность retrieval-пайплайна.

---

### Dense Retrieval

Для семантического поиска используется embedding-модель:

```text
ai-forever/ru-en-RoSBERTa
```

Каждый чанк преобразуется в плотный вектор.

Для поиска ближайших документов используется **FAISS**.

Dense Retrieval позволяет находить:

* семантически похожие документы;
* перефразированные формулировки;
* тематически связанные фрагменты;
* ответы без полного совпадения ключевых слов.

---

### Sparse Retrieval

Для лексического поиска используется **TF-IDF** из библиотеки Scikit-learn.

TF-IDF учитывает совпадения слов между запросом и документами и особенно полезен для поиска:

* названий продуктов;
* терминов;
* аббревиатур;
* числовых значений;
* точных формулировок.

---

### Hybrid Search

Результаты FAISS и TF-IDF объединяются в единую оценку релевантности.

```text
hybrid_score =
    alpha × dense_score
    + (1 - alpha) × sparse_score
```

где:

* `dense_score` — оценка семантической близости;
* `sparse_score` — оценка TF-IDF;
* `alpha` — вес семантического поиска.

Гибридный подход компенсирует ограничения отдельных методов:

| Метод         | Преимущество                | Ограничение                     |
| ------------- | --------------------------- | ------------------------------- |
| FAISS         | Находит документы по смыслу | Может пропускать точные термины |
| TF-IDF        | Находит точные совпадения   | Не понимает перефразирование    |
| Hybrid Search | Объединяет оба сигнала      | Требует настройки веса `alpha`  |

---

### Cross-Encoder Reranking

После гибридного поиска лучшие кандидаты передаются Cross-Encoder модели:

```text
DiTy/cross-encoder-russian-msmarco
```

Cross-Encoder получает пару:

```text
(query, chunk)
```

и вычисляет итоговую оценку релевантности.

В отличие от поиска по отдельным эмбеддингам, Cross-Encoder анализирует запрос и документ совместно.

Это позволяет:

* улучшить порядок найденных документов;
* удалить менее релевантные результаты;
* повысить качество Top-K;
* передать LLM более точный контекст.

---

### Answer Generation

После reranking выбираются лучшие чанки, которые формируют контекст для языковой модели.

```text
Top-K chunks → Context → LLM → Answer
```

Ответ формируется на основе найденной информации из корпуса документов.

## Оценка качества

В проекте предусмотрен модуль оценки retrieval-пайплайна.

Для анализа поиска могут использоваться следующие метрики.

### Hit@K

Показывает, присутствует ли хотя бы один релевантный документ среди первых `K` результатов.

```text
Hit@K = 1, если релевантный документ найден в Top-K
Hit@K = 0, если релевантный документ не найден
```

### Recall@K

Показывает, какая доля релевантных документов попала в Top-K.

```text
Recall@K =
    количество найденных релевантных документов
    /
    общее количество релевантных документов
```

### Precision@K

Показывает долю релевантных документов среди первых `K` результатов.

```text
Precision@K =
    количество релевантных документов в Top-K
    /
    K
```

### MRR

Оценивает позицию первого релевантного документа.

```text
MRR =
    mean(1 / rank)
```

где `rank` — позиция первого релевантного результата.

## Почему используется гибридный поиск

Семантический поиск хорошо работает с перефразированными запросами, но может пропускать точные термины.

TF-IDF хорошо находит совпадения слов, но не учитывает общий смысл запроса.

Например, запрос:

```text
Льготный период по карте Alfa Travel
```

может содержать одновременно:

* точное название продукта;
* финансовый термин;
* смысловую формулировку вопроса.

FAISS находит документы по смыслу, а TF-IDF сохраняет чувствительность к точным словам.

---

## Почему reranking выполняется отдельно

На первом этапе требуется быстро найти кандидатов среди большого количества документов.

Для этого используются FAISS и TF-IDF.

На втором этапе Cross-Encoder более точно оценивает ограниченный набор найденных чанков.

```text
Fast Retrieval → Accurate Reranking
```

Это позволяет сохранить приемлемую скорость поиска и повысить качество финальной выдачи.

---

## Ограничения

* качество поиска зависит от выбранной стратегии chunking;
* Cross-Encoder увеличивает время обработки запроса;
* первый запуск требует загрузки моделей;
* итоговое качество зависит от состава и качества исходного корпуса;
* параметры гибридного поиска необходимо настраивать на тестовой выборке.
